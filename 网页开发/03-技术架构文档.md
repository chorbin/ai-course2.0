## 1. 技术选型说明

### 1.1 核心原则

本项目采用**纯前端架构**，遵循以下原则：

| 原则 | 说明 | 理由 |
|------|------|------|
| 零后端依赖 | 无需服务器部署 | 简化运维，降低成本，可直接部署到静态托管平台 |
| 本地运行 | 可直接本地打开 | 用户无需联网即可使用，便于分发 |
| 轻量快速 | 最小化依赖 | 快速加载，无需复杂构建流程 |
| 易于维护 | 结构清晰 | 非技术人员也能理解和修改 |

#### 1.2.2 具体选型

| 层级 | 技术 | 版本 | 选择理由 |
|------|------|------|----------|
| **Markdown渲染** | marked.js | v12+ | 轻量(~40KB)、高性能、插件生态丰富、支持扩展 |
| **代码高亮** | highlight.js | v11+ | 支持190+语言、自动检测、主题丰富 |
| **图表渲染** | Mermaid | v10+ | 流程图/时序图/甘特图原生支持，与Markdown无缝集成 |
| **样式框架** | 原生CSS + CSS变量 | - | 零依赖、完全可控、便于主题切换 |
| **交互逻辑** | 原生JavaScript (ES6+) | - | 无框架负担、性能最优、学习成本低 |
| **数学公式** | KaTeX | v0.16+ | 比MathJax快100倍、轻量级 |

#### 1.2.3 备选方案对比

**Markdown渲染引擎对比：**

| 方案 | 大小 | 性能 | 功能 | 推荐度 |
|------|------|------|------|--------|
| marked.js | ~40KB | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ✅ 推荐 |
| markdown-it | ~150KB | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 备选 |
| showdown | ~60KB | ⭐⭐⭐ | ⭐⭐⭐ | 不推荐 |
| CommonMark | ~100KB | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 备选 |

**为什么不使用前端框架？**

| 框架 | 大小 | 适用场景 | 本项目适用性 |
|------|------|----------|--------------|
| React | ~130KB | 复杂交互应用 | ❌ 过度设计 |
| Vue | ~100KB | 渐进式增强 | ❌ 非必要 |
| Svelte | ~20KB | 编译时优化 | ⚠️ 需要构建 |
| 原生JS | 0KB | 简单内容展示 | ✅ 最佳选择 |

### 4.4 文件加载策略

```
┌─────────────────────────────────────────────────────────────────────┐
│                       文件加载策略                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  加载时机:                                                         │
│                                                                     │
│  1. 按需加载 (Lazy Loading)                                        │
│     ┌─────────────────────────────────────────────────────┐       │
│     │ 用户点击章节 → 加载该章节MD文件                       │       │
│     │ 优点: 初始加载快，内存占用小                          │       │
│     │ 缺点: 首次切换有延迟                                  │       │
│     └─────────────────────────────────────────────────────┘       │
│                                                                     │
│  2. 预加载 (Preloading)                                            │
│     ┌─────────────────────────────────────────────────────┐       │
│     │ 当前章节加载完成 → 后台预加载下一章节                  │       │
│     │ 优点: 切换无感知                                      │       │
│     │ 缺点: 可能加载用户不看的章节                          │       │
│     └─────────────────────────────────────────────────────┘       │
│                                                                     │
│  3. 缓存策略 (Caching)                                             │
│     ┌─────────────────────────────────────────────────────┐       │
│     │ 内存缓存: 已加载的章节保存在 Map 中                    │       │
│     │ 持久缓存: 可选使用 IndexedDB 离线存储                 │       │
│     └─────────────────────────────────────────────────────┘       │
│                                                                     │
│  推荐方案:                                                         │
│  ┌─────────────────────────────────────────────────────┐          │
│  │ 按需加载 + 预加载下一章 + 内存缓存                      │          │
│  │                                                      │          │
│  │ 首次访问:                                            │          │
│  │   1. 加载目录元数据 (toc.json) ~2KB                  │          │
│  │   2. 加载首页章节                                    │          │
│  │   3. 预加载第2章                                     │          │
│  │                                                      │          │
│  │ 后续导航:                                            │          │
│  │   1. 检查缓存                                        │          │
│  │   2. 命中 → 直接渲染                                 │          │
│  │   3. 未命中 → fetch加载 → 缓存 → 渲染                │          │
│  │   4. 预加载下一章                                    │          │
│  └─────────────────────────────────────────────────────┘          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
